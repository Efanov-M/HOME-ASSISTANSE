1. first()

self.db.query(User).filter(User.email == email).first()

Что делает

Берёт первую найденную запись.

Что вернёт
 • объект ORM-модели
 • или None, если ничего нет

Когда использовать

Когда:
 • нужна одна запись
 • и тебе не страшно, что их теоретически может быть несколько

⸻

1. scalar()

self.db.query(User.id).filter(User.email == email).scalar()

Что делает

Возвращает одно скалярное значение.

Скалярное = одно простое значение:
 • число
 • строка
 • дата

Что вернёт
 • например 5
 • или None

Когда использовать

Когда ты выбираешь одну колонку, а не весь объект.

⸻

1. all()

self.db.query(User).all()

Что делает

Возвращает все найденные записи

Что вернёт

Список:

[User(...), User(...), User(...)]

или пустой список:

[]

Когда использовать

Когда нужно получить много объектов.

Например:
 • всех пользователей семьи
 • все audit log записи пользователя

⸻

1. one()

self.db.query(User).filter(User.email == email).one()

Что делает

Требует, чтобы результат был ровно один

Что будет
 • если запись одна → вернёт её
 • если записей нет → ошибка
 • если записей больше одной → ошибка

Когда использовать

Когда ты уверен, что должна быть ровно одна запись, и хочешь жёсткое поведение.

⸻

1. one_or_none()

self.db.query(User).filter(User.email == email).one_or_none()

Что делает

Ожидает:
 • либо одну запись
 • либо ни одной

Что будет
 • одна запись → вернёт объект
 • нет записей → None
 • больше одной → ошибка

Когда использовать

Когда запись должна быть уникальной, и ты хочешь это контролировать.

Для email, например, это часто хороший вариант, потому что email у тебя уникален.

⸻

1. count()

self.db.query(User).count()

Что делает

Возвращает количество записей

Что вернёт

Число:

5

Когда использовать

Когда нужно узнать, сколько строк найдено.

Например:
 • сколько пользователей в семье
 • сколько активных токенов у пользователя

⸻

1. scalar_one() и scalar_one_or_none()

Это уже более новый и строгий стиль, чаще используется в SQLAlchemy 2.x через select(). Пока тебе рано в это углубляться.
Для текущего этапа тебе достаточно понять:
 • first()
 • scalar()
 • all()
 • one_or_none()

⸻

Самая понятная таблица

Метод Возвращает Если нет записей Если много записей
first() первая запись None просто возьмёт первую
scalar() одно значение None зависит от запроса
all() список [] список всех
one() ровно одну запись ошибка ошибка
one_or_none() одну запись или None None ошибка
count() число 0 число

⸻

На одном живом примере

Допустим, у тебя есть таблица пользователей.

⸻

Получить одного пользователя

user = self.db.query(User).filter(User.email == email).first()

Результат
 • объект User
 • или None

⸻

Получить только его id

user_id = self.db.query(User.id).filter(User.email == email).scalar()

Результат
 • 1
 • или None

⸻

Получить всех пользователей семьи

users = self.db.query(User).filter(User.family_id == family_id).all()

Результат
 • список объектов User

⸻

Жёстко проверить, что email уникален

user = self.db.query(User).filter(User.email == email).one_or_none()

Результат
 • один пользователь
 • или None
 • если в БД бардак и email дублируется — ошибка

⸻

Что тебе лучше использовать сейчас

Для твоего текущего уровня и проекта:

чаще всего:
 • first()
 • all()

иногда:
 • scalar()

реже, но полезно:
 • one_or_none()

⸻

Практическое правило для тебя

Нужен объект → first()

query(User)...first()

Нужен список объектов → all()

query(User)...all()

Нужно одно поле → scalar()

query(User.id)...scalar()

Нужна строгая уникальность → one_or_none()

query(User)...one_or_none()
