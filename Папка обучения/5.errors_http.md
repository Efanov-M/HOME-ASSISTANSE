
# Это и будет единая политика HTTP error handling для текущего auth-ядра

## • 400 Bad Request

 • ошибки валидации входных данных
 • некорректный email
 • слабый пароль
 • пустые/некорректные значения
 • прочие ValueError, связанные с форматом и содержимым входа

## • 401 Unauthorized

 • AccessTokenNotFound
 • RefreshTokenNotFound
 • RefreshTokenRevoked
 • RefreshTokenExpired
 • RefreshTokenNoExpiry
 • ResetTokenNotFound
 • ResetTokenExpired
 • ResetTokenUsed
 • ResetTokenNoExpiry
 • ValueError("Неверный пароль")

## • 403 Forbidden

 • RefreshUserBlocked
 • AccessUserBlocked

## • 404 Not Found

 • RefreshUserNotFound
 • AccessUserNotFound

## • 409 Conflict

 • ValueError("Пользователь с таким email уже существует")
 • ValueError("Семья с таким именем уже существует")
 • Правило реализации
 • AuthError и ValueError обрабатываются единообразно
 • базовый механизм — global exception handlers в main.py
 • локальные HTTPException остаются только там, где это чисто HTTP-ошибка endpoint, например:
 • отсутствует Authorization header
 • неверный формат Authorization header
