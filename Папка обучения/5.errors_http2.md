
Нужно делать global exception handlers в main.py.
Это закроет проблему 500 на штатных auth-ошибках и не засорит api/auth.py.

 Что такое global exception handler:
 • это общий обработчик, который FastAPI вызывает, если из endpoint или service “вылетело” определённое исключение
 • он превращает Python-исключение в нормальный HTTP-ответ

 Почему это лучше, чем try/except в каждом endpoint:
 • один механизм на всё приложение
 • меньше копипаста
 • правила ошибок хранятся в одном месте
 • проще менять политику статусов позже

 Что именно тебе нужно сделать в main.py:
 • оставить создание FastAPI()
 • оставить подключение auth_router
 • добавить обработчик для AuthError
 • добавить обработчик для ValueError

 Какой должна быть логика обработчика AuthError:
 • он получает объект ошибки
 • смотрит на её класс
 • по зафиксированной таблице выбирает HTTP-статус
 • возвращает JSON-ответ с сообщением ошибки

 Что он должен возвращать по форме:
 • обычный JSON-объект
 • минимально достаточно поля:
 • detail
 • то есть по смыслу ответ должен быть вида:
 • статус HTTP
 • текст ошибки из exception.message

 Как должна работать логика маппинга для AuthError:
 • если ошибка из группы токенов → 401
 • если ошибка из группы blocked user → 403
 • если ошибка из группы user not found → 404
 • Что делать с ValueError:
 • нужен отдельный обработчик
 • он смотрит на текст ошибки
 • если это конфликт существующего ресурса:
 • “Пользователь с таким email уже существует”
 • “Семья с таким именем уже существует”
 • тогда 409
 • если это “Неверный пароль”:
 • тогда 401
 • всё остальное из ValueError:
 • 400

 Почему ValueError обрабатывается хуже, чем AuthError:
 • у AuthError есть структура и типы классов
 • у ValueError сейчас только текст
 • значит маппинг по ValueError будет менее строгим
 • но для текущего этапа это допустимо

 Что важно не сломать:
 • не убирать локальные HTTPException из /auth/me
 • ошибки вроде:
 • отсутствует Authorization
 • неправильный формат заголовка
 • неправильная схема Bearer
уже нормально живут прямо в endpoint
 • global handlers нужны именно для ошибок из service-слоя
 • Что должно измениться после внедрения:
 • неверный логин больше не даёт 500
 • просроченный refresh token больше не даёт 500
 • reset token error больше не даёт 500
 • blocked user больше не даёт 500
