
🎯 ШАГ 0 — Архитектура запуска (Docker-first)

Ты будешь запускать систему как набор сервисов.
Это уже мини-инфраструктура, а не просто приложение.

⸻

🧱 1. Состав системы (контейнеры)

Минимальный набор:

Обязательные сервисы
 1. backend
 • FastAPI приложение
 • бизнес-логика
 2. database
 • PostgreSQL
 • хранение данных
 3. reverse proxy
 • HTTPS
 • доступ из интернета
 • (рекомендую Caddy)

⸻

Дополнительно (но закладываем сразу)
 4. storage (volume)
 • файлы документов
 5. backup service (в будущем)
 • резервные копии БД и файлов

⸻

🧠 Итоговая схема

Internet
   ↓
[ Reverse Proxy (HTTPS) ]
   ↓
[ Backend (FastAPI) ]
   ↓
[ PostgreSQL ]

+ File Storage (volume)

⸻

📁 2. Структура проекта (очень важно)

Ты сейчас закладываешь скелет на месяцы вперёд.

Рекомендуемая структура

home-secretary/
│
├── backend/
│   ├── app/
│   │   ├── core/          # конфиг, безопасность
│   │   ├── models/        # модели БД
│   │   ├── api/           # роуты
│   │   ├── services/      # бизнес-логика
│   │   ├── repositories/  # работа с БД
│   │   └── modules/       # tasks, finance, docs (потом)
│   │
│   ├── requirements.txt
│   └── Dockerfile
│
├── docker/
│   ├── docker-compose.yml
│   └── .env
│
├── storage/
│   └── uploads/           # файлы пользователей
│
├── README.md
└── .gitignore

⸻

🧠 Почему такая структура

Это не “красиво”, а архитектурно правильно:

Папка Назначение
core безопасность, настройки
models структура БД
api входные точки
services логика
repositories работа с БД

👉 Ты сразу разделяешь:
 • что приходит (API)
 • что делает (service)
 • как хранится (repository)

⸻

⚠️ Типичная ошибка новичка

❌ писать всё в одном файле
❌ мешать логику и БД
❌ не разделять слои

👉 потом это невозможно масштабировать

⸻

🐳 3. Docker архитектура (логика)

Ты НЕ пишешь пока compose, ты понимаешь:

Сервис backend
 • билдится из Dockerfile
 • зависит от PostgreSQL
 • получает настройки через .env

⸻

Сервис database
 • postgres:latest
 • volume для данных

⸻

Сервис proxy
 • принимает HTTPS
 • проксирует на backend

⸻

🔐 4. Переменные окружения (важно сразу)

Ты должен сразу привыкать:

никаких хардкодов

Примеры (без кода):
 • DATABASE_URL
 • JWT_SECRET
 • POSTGRES_USER
 • POSTGRES_PASSWORD
 • POSTGRES_DB

⸻

💾 5. Хранение данных

База
 • PostgreSQL volume

Файлы
 • storage/uploads/

👉 это важно:
файлы НЕ в контейнере, а в volume

⸻

⚠️ 6. Критические решения (фиксируем сейчас)

⸻

6.1. Backend = один сервис
 • НЕ микросервисы
 • НЕ разделение на 5 контейнеров

👉 один backend

⸻

6.2. БД = отдельный контейнер

👉 обязательно

⸻

6.3. Файлы = вне контейнера

👉 иначе потеряешь данные при пересборке

⸻

6.4. Proxy обязателен

Почему:
 • HTTPS
 • домен
 • безопасность
